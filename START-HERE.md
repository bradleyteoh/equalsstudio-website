# Editing the Equals Design Studio website — start here

Hi! This folder is the Equals Design Studio website. You can add and edit content here (like adding our portfolio work) **without knowing any code** — you'll use an assistant called **Claude Code** to do it, and you just describe what you want in plain English.

**Important:** this website publishes itself. When you save and push your changes, the live equalsstudio.com updates automatically about a minute later. So **always look at the preview first** and make sure you're happy before you publish. If you just want to experiment, don't publish — you can look at the preview as much as you like without anything going live.

## One-time setup

1. **Install Claude Code** — follow the steps at https://claude.com/claude-code . Bradley can help with this part.
2. **Open this folder in Claude Code.** In the Terminal app, type `cd ` (with a space), drag this folder onto the window, press Enter, then type `claude` and press Enter.

That's it — you're in.

## How to add portfolio work (or edit anything)

Just tell Claude Code what you want, for example:

> "Add a Portfolio section to the site with these 4 projects. For each one I'll give you a photo, the project name, and a one-line description."

or

> "Add a new portfolio project: name is 'Sunway Office Fit-Out', here's the photo, description is 'A 4,000 sq ft corporate office in Bandar Sunway...'"

Then, to see how it looks:

> "Preview the site."

Claude Code will give you a link (like http://localhost:8000) — open it in your browser to see your changes live.

**Photos:** put project photos anywhere easy to find, and just tell Claude Code where they are (you can drag a photo into the Claude Code window). It will place them correctly.

## When you're happy — publishing it live

Look at the preview one more time. If it looks right, tell Claude Code:

> "Publish my changes."

It will save everything and push it, and the live equalsstudio.com will update by itself in about a minute. That's it — you published it.

**Only publish when you've checked the preview and you're happy.** Once you publish, it's live for everyone. (If you're not sure yet, just don't say "publish" — keep looking at the preview.)

## A few things to know

- **Publishing is a deliberate step.** Nothing goes live until you say "publish." Previewing is always safe.
- **If something looks wrong,** just tell Claude Code "undo that" or describe what you'd like different. It keeps a history, and even a published change can be rolled back — ask Bradley if you need that.
- **Stuck?** Message Bradley — he set this up.
